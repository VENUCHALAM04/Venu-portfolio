Project Specification Sheet

Smart Black Box for Accident Analysis

System Design Objective:

This safety system continuously monitors vehicular parameters like tilt angle and extreme impact G-forces. On detecting an accident, the ESP32 commands a camera module to capture a snapshot, logs evidence to Firebase, and dispatches a critical SMS notification containing GPS navigation coordinates.

Engineered Core Features

  • Automatic accident impact detection via IMU / Accelerometer thresholding
  • Instant SMS alert routing through an API/cellular gateway with exact GPS coordinates
  • Real-time cloud database storage (Firebase) for post-crash collision analysis
  • Web-based administration dashboard tracking vehicle telemetry speeds and orientation

Critical Challenges & Mitigations

  • Synchronizing high-speed sensor readings to prevent false positive crash detections.
  • Achieving low latency in uploading camera snapshots over cellular networks during emergency intervals.

Resonance Assistive Feedback Device

System Design Objective:

Designed as a wearable tech device, it converts environmental cues and obstacles into structured high-frequency auditory soundscapes and variable haptic vibrating pulses. This feedback enables users to safely navigate their surroundings and feel responsive signals.

Engineered Core Features

  • Ultrasonic distance ranging mapped dynamically to haptic motor speeds
  • Distinct auditory tone profiles for various threat vectors and ranges
  • Comfortable wearable casing optimized for hand or chest mounting
  • Ultra-low power sleep states maximizing operational device cycles

Critical Challenges & Mitigations

  • Calibrating multiple sensors in close proximity to prevent spatial signal echo overlaps.
  • Designing an intuitive feedback language that users can understand without cognitive fatigue.

Courier Management System

System Design Objective:

Developed to replace manual ledger tracking, this client-server system manages package inventories, bookings, customer profiles, and staff status assignments. It provides users with a clean shipment tracking ledger while granting admins a secure database command center.

Engineered Core Features

  • Role-based secure authentication portals (Customer, Admin, Courier Staff)
  • Dynamic dispatch booking invoice generation and billing math
  • Interactive package status history (Booked -> In Transit -> Hub -> Out for Delivery -> Delivered)
  • Comprehensive SQL reporting queries showing transit metrics and load averages

Critical Challenges & Mitigations

  • Structuring relational database tables with proper foreign keys to avoid redundant records during tracking updates.
  • Safeguarding user inputs against SQL injection and common session hijacking vulnerabilities.
